CSS Units

Many CSS properties like width, margin, padding, and font-size take a length, and CSS has many different ways to express length.CSS units can be of two types: absolute and relative.
Absolute units are normally considered of same size and are not relative to anything. cm, mm, in, px are some of commonly used absolute units.
Relative units on the other hand, are dependent on something else, generally, size of the element or parent element, they change to change their size based on other factor. em, vh, vw, rem are some commonly used relative units . Some of commonly used css units are:

1. px

Definition

Pixels or simply px are one of the most common length units in CSS. In CSS, 1 pixel is formally defined as 1/96 of an inch.

.paragraph{
font-size : 16px
}

Examples

This example uses 16px of font size. This is the default font-size for most of the browsers

2. em

Definition

The CSS em unit gets its name from a typographical unit. In typography, the term em was originally a reference to the width of the capital M in the typeface and size being used. When used with the font-size property, em inherits the font-size from its parent element.

.paragraph{
font-size : 1.5em
}

Examples

I am parent with 30px of font-size

I am using font size of 1.5em. My font-size will depend on my parent and I am 1.5 times the size of my parent.

3. rem

Definition

This is simply known as root em. This relative unit is not affected by the size or setting of a parent element, and is instead based on the root of the document. For websites, the root of the document is the html element. In most browsers, the default font size is 16, so the font-size of html elements is 16px. But if a user changes their browser's default font size, then the font-size of p will scale up or down accordingly.

.paragraph{
font-size : 1.5rem
}

Examples

I am parent with 30px of font-size

I am using font size of 1.5rem. My font-size will depend on the size of root element and not my parent and I am 1.5 times the size of root element.

4. percentage (%)

Definition

Percentages, are relative to the parent's size. For example, if you set an element's font-size as a percentage, it will be a percentage of the font-size of the element's parent.

.paragraph{
font-size : 120%
}

Examples

I am parent with 30px of font-size

I am using font size of 120%. My font-size will depend on the size of my parent.

I am also using font size of 120%. But, my font-size will depend on the size of root element.

4. vw

Definition

vw, also commonly known as view width, is relative to viewport width. 1vw is 1% of the width of the viewport.

.container{
height : 100vw
}

Examples

I am using 100vw. I will take up the full width available. So if you resize your browser to 690 pixels wide, then I will take up all 690 pixels in width.

5. vh

Definition

vh, also commonly known as view height, is relative to viewport height. 1vh is 1% of the height of the viewport.

.container{
height : 100vh
}

Examples

I am using 100vh. I will take up the 100% of height available. So if the browser window is 900 pixels high, my height will be 900 pixels.

6. dvh

Definition

They were introduced, when mobile browsers started changing viewport height as user scrolls down or up to see more content. The traditional, vh , were not behaving correctly when view height was changing. The dynamic view port height (dvh), are defined with respect to any viewport size, that are dynamically expanded or retracted.
Let's say you have a phone that has 100px of screen space. When you load up a page, the browser UI takes up 15px which leaves 85px left for your website's content. However, browser, when you scroll down the browser UI only takes up 10px, which leaves your website content with 90px.

.container{
height : 100dvh
}

Examples

I am using 100vh. I will take up exactly 100% of height available on any screen. See me on mobile device to feel difference
